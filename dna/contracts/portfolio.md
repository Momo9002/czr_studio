# CZR Portfolio Contract

> Source: `dna/identity.json`, `dna/visual.md`, `dna/voice.md`
> Apply this to every case study page and every Instagram portfolio post.

---

## Case Study Page Requirements

### Structure
1. **Hero** — Dark. Full-bleed client site screenshot or video. Client name in Syne 800.
2. **Context** — 2-3 sentence brief. What the client does, where, what they needed.
3. **Delivery** — Time delivered, pages built.
4. **Screenshots** — Light section. Minimum 3: desktop hero, mobile view, key inner page.
5. **Knight watermark** — Bottom-right, `♞ CZR`, 30% opacity.

### URL Pattern
`czr.studio/cases/[client-slug]/`

### Meta
```html
<title>[Client] — Built by CZR Studio in 48h</title>
<meta name="description" content="[Client]. [One sentence]. Delivered in [X] hours by CZR Studio.">
```

## Instagram Portfolio Post

Image: device mockup (see `visual.md` for dark vs light rules)

Caption format (from `voice.md`):
```
[Project Name]. [City / Industry].

[One-sentence result].
Delivered in [X] hours.

→ czr.studio

#CZRStudio #HauteCoutureDigital [2 relevant tags]
```

## Quality Gate

A portfolio post only goes live if:
- The image looks like it belongs in AD Architectural Digest
- The device shows real CZR work (not a mockup of fake content)
- The caption follows the template exactly
- The post fits the current grid mode (dark or light alternation)
